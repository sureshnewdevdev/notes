Overview

This guide explains how to deploy an MCP application from VS Code to Azure App Service on Windows.

Important: Azure App Service is best suited for web/API applications. If your MCP server is only STDIO-based, convert it to an HTTP-based MCP server before deploying to App Service.

For cloud deployment, expose your MCP tools through an HTTP endpoint such as:

https://your-mcp-app.azurewebsites.net/mcp

Architecture

VS Code | | Deploy v Azure App Service - Windows | | Public HTTPS URL v MCP HTTP Server | | Calls tools / APIs / database v Your backend logic

Example final URL:

https://your-mcp-app.azurewebsites.net

Possible MCP endpoint:

https://your-mcp-app.azurewebsites.net/mcp

Part 1: Prepare MCP App Locally

Step 1: Create Project Folder

mkdir my-mcp-azure-app
cd my-mcp-azure-app

Step 2: Create Virtual Environment

python -m venv .venv

Step 3: Activate Virtual Environment

.venv\Scripts\activate

Step 4: Install Required Packages

pip install fastapi uvicorn
pip install mcp

Step 5: Create requirements.txt

pip freeze > requirements.txt

Part 2: Create Simple MCP-style HTTP App

Create a file named main.py.

from fastapi import FastAPI
from pydantic import BaseModel
from datetime import datetime

app = FastAPI(title="Azure MCP Demo Server")


class ToolRequest(BaseModel):
    tool_name: str
    input: str | None = None


@app.get("/")
def home():
    return {
        "message": "MCP Server is running on Azure App Service",
        "status": "ok",
        "endpoints": ["/health", "/tools", "/mcp"]
    }


@app.get("/health")
def health():
    return {
        "status": "healthy",
        "time": datetime.utcnow().isoformat()
    }


@app.get("/tools")
def list_tools():
    return {
        "tools": [
            {
                "name": "get_current_time",
                "description": "Returns current server time"
            },
            {
                "name": "reverse_text",
                "description": "Reverses the given text"
            }
        ]
    }


@app.post("/mcp")
def call_tool(request: ToolRequest):
    if request.tool_name == "get_current_time":
        return {
            "tool": "get_current_time",
            "result": datetime.utcnow().isoformat()
        }

    if request.tool_name == "reverse_text":
        text = request.input or ""
        return {
            "tool": "reverse_text",
            "input": text,
            "result": text[::-1]
        }

    return {
        "error": "Unknown tool",
        "available_tools": ["get_current_time", "reverse_text"]
    }

Part 3: Test Locally

Run the app locally:

uvicorn main:app --host 127.0.0.1 --port 8000

Open these URLs in your browser:

http://127.0.0.1:8000
http://127.0.0.1:8000/health
http://127.0.0.1:8000/docs

Test MCP endpoint using PowerShell:

Invoke-RestMethod `
  -Uri "http://127.0.0.1:8000/mcp" `
  -Method POST `
  -ContentType "application/json" `
  -Body '{"tool_name":"reverse_text","input":"Hello MCP"}'

Expected result:

{
  "tool": "reverse_text",
  "input": "Hello MCP",
  "result": "PCM olleH"
}

Part 4: Prepare Files for Azure App Service

Your folder should look like this:

my-mcp-azure-app
│
├── main.py
├── requirements.txt
└── .venv
Do not deploy .venv to Azure App Service.

Create a .gitignore file:

.venv/
__pycache__/
*.pyc
.env

Install these VS Code extensions:

Part 6: Create Azure App Service - Windows

In VS Code Azure panel:

Azure → App Service → Create New Web App
Setting Recommended Value

RegionCentral India / South India / East US

For production MCP/API apps, use Basic B1 or above because Always On is usually needed.

Part 7: Deploy from VS Code

  1. Open the Azure panel in VS Code.
  2. Right-click your App Service.
  3. Click Deploy to Web App.
  4. Select your project folder: my-mcp-azure-app.
  5. Confirm deployment.

VS Code will zip your project files and deploy them to Azure App Service.

Part 8: Configure Startup Command

This is the most important step for Python FastAPI apps on Windows App Service.

Go to Azure Portal:

Azure Portal → App Services → Your App → Configuration → General settings

Set the startup command:

python -m uvicorn main:app --host 0.0.0.0 --port %HTTP_PLATFORM_PORT%

Then click:

Save → Restart

Why use %HTTP_PLATFORM_PORT%?

On Windows App Service, Azure assigns a dynamic port. Your app should listen on that port instead of hardcoding port 8000.

Add App Settings

Go to:

App Service → Configuration → Application settings

Add:

SCM_DO_BUILD_DURING_DEPLOYMENT = true

Optional:

PYTHON_ENABLE_WORKER_EXTENSIONS = 1

If your app uses secrets, add them as Application Settings:

API_KEY = your-secret-key
DATABASE_URL = your-db-connection-string
Do not hardcode secrets inside Python files.

Part 10: Enable Logs

Go to:

App Service → Monitoring → App Service logs

Enable:

  • Application Logging
  • Web Server Logging
  • Detailed Error Messages
  • Failed Request Tracing

Then open:

App Service → Log stream

This helps you see startup errors, missing packages, wrong file names, or port issues.

Part 11: Common Errors and Fixes

Error 1: Default Azure Page Showing

Meaning: Your files deployed, but Azure did not start your Python app correctly.

Fix:

python -m uvicorn main:app --host 0.0.0.0 --port %HTTP_PLATFORM_PORT%

Also check that your file is named main.py and your FastAPI variable is app.

Error 2: Application Error

Check Log Stream. Common reasons:

  • requirements.txt missing
  • uvicorn not installed
  • Wrong startup command
  • Wrong Python version
  • main.py missing
  • app variable name mismatch

Error 3: No module named uvicorn

pip install uvicorn fastapi
pip freeze > requirements.txt

Then redeploy.

Error 4: Could not import module "main"

This means Azure cannot find main.py.

If your file is in project root, use:

python -m uvicorn main:app --host 0.0.0.0 --port %HTTP_PLATFORM_PORT%

If your file is inside a folder like app/main.py, use:

python -m uvicorn app.main:app --host 0.0.0.0 --port %HTTP_PLATFORM_PORT%

Error 5: Port Issue

Do not hardcode this in Azure:

--port 8000

Use this instead:

--port %HTTP_PLATFORM_PORT%

Part 12: Connect This Azure MCP App to Copilot Studio

Once deployed, your public endpoint is:

https://your-app-name.azurewebsites.net/mcp

Basic integration flow:

Copilot Studio Agent ↓ Action / Connector / Power Automate ↓ HTTP POST ↓ Azure App Service MCP endpoint ↓ Python tool logic ↓ Response back to Copilot

Example request body:

{
  "tool_name": "reverse_text",
  "input": "Copilot Studio"
}

Example response:

{
  "tool": "reverse_text",
  "input": "Copilot Studio",
  "result": "oidutS tolipoc"
}
